팝업 2단계: ID 및 위치 설정

데이터베이스 이름과 서버의 위치를 정하는 곳입니다. 손대지 말고 그대로 두고 진행하셔도 무방합니다.

  • 데이터베이스 ID: 회색 글씨로 (default)라고 적혀 있는 상태 그대로 둡니다.
  • 위치: nam5 (United States) 또는 기본으로 선택된 위치를 그대로 둡니다.

팝업 3단계: 구성 (⭐매우 중요⭐)

데이터를 연동하기 위해 가장 중요한 단계입니다. 반드시 아래의 옵션을 체크해야 합니다!

테스트 모드에서 시작
빠른 설정을 위해 기본적으로 데이터가 공개됩니다. 하지만 30일 이내에 보안 규칙을 업데이트하여...

체크를 확인했다면 바닥의 파란색 만들기 버튼을 꾹 누르세요!

※ 화면에 파란 동그라미가 돌면서 'Cloud Firestore 프로비저닝 중...' 이라는 문구가 나오면 정상적으로 창고가 지어지고 있는 것입니다. 잠시 기다려주세요.

4

팝업 4단계: 규칙 영구 개방 (if true)

생성이 완료되면 위쪽 탭에서 [규칙]을 누릅니다. 30일 제한 타이머 코드를 지우고, 영원히 데이터를 연동할 수 있도록 코드를 수정합니다.

match /{document=**} {
  allow read, write: if true;
}

수정 후 반드시 우측의 파란색 [게시] 버튼을 누르세요!

※ '게시된 변경사항이 반영되는 데 최대 1분이 소요될 수 있습니다.' 메시지가 뜨면 완벽하게 성공한 것입니다.

앱 이름 짓고 열쇠(코드) 복사하기

이제 웹 앱의 이름을 지어주고, 내 컴퓨터와 연결할 코드를 발급받습니다.

Firebase 호스팅 설정 (무시하세요)
// Import the functions you need from the SDKs you need
import { initializeApp } from "firebase/app";

✂️
드래그해서 복사!
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
// Initialize Firebase
const app = initializeApp(firebaseConfig);
  • '앱 닉네임' 칸에 아무 이름이나 (예: my-web) 적어줍니다.
  • 아래의 파란색 [앱 등록] 버튼을 누릅니다.
  • 밑에 코드가 나타나면, const firebaseConfig = { ... } 모양의 코드를 찾아 딱 그 덩어리 부분만 전부 드래그해서 복사(Ctrl+C)합니다!

PART 2

내 HTML 파일에 열쇠 넣기

복사한 열쇠 붙여넣고 저장

내 컴퓨터에 있는 엑셀용 HTML 파일을 열고, 아까 파이어베이스에서 복사해 둔 열쇠 코드를 아무 데나 넣으면 안 됩니다! 반드시 코드 아래쪽의 <script> 태그 영역 안에 붙여넣어야 합니다.

<body>
  ... (화면 구성 내용) ...

  <script type="module">
    // 👇 복사한 코드를 여기에 붙여넣습니다 (Ctrl+V)
    const firebaseConfig = { ... };
    const app = initializeApp(firebaseConfig);
  </script>
</body>

코드를 알맞게 넣었다면 반드시 저장(Ctrl+S)을 눌러줍니다.
